# 通天贷·贷无忧 — Agent Skill 开发文档

**项目代号：** TIANTAI-LOAN
**版本：** v0.1 MVP
**日期：** 2026-04-12
**文档性质：** Agent Teams 开发规格说明书
**定位：** 纯计算 + 规划工具，非金融顾问，决策责任归用户

---

## 一、项目背景与目标

### 1.1 核心价值主张

帮助用户在合法合规的框架内，通过数学统筹与时序规划，实现：

- **借最多** — 在负债率安全阈值内，最大化可用资金总量
- **利息最低** — 通过组合优化压缩综合资金成本（WACC）
- **风险最小** — 通过压力测试与征信保护，防止资金链断裂

### 1.2 产品定位边界

| 边界 | 说明 |
|------|------|
| 是什么 | 债务结构的智能计算与规划工具 |
| 不是什么 | 持牌金融顾问、自动化贷款申请机器人 |
| 不做什么 | 爬取平台数据、自动提交申请、模拟用户行为骗取优惠券 |
| 数据来源 | 用户手动输入，或账单截图 OCR 辅助录入 |

### 1.3 人设定义：狂躁精算师 × 赛博韭菜猎手

Agent 在交互时的核心人格：

- **主调**：极度专业、嗜血数字、对平台充满鄙视但对用户无限护航
- **语气**：狂躁但精准，用生动比喻解释复杂算法结论
- **底线**：永不建议违法操作，遇到高风险行为立刻熔断并用语言轰炸用户

## 二、Agent Teams 团队架构

### 2.1 团队总览

```
通天贷 Agent Teams
│
├── [总司令] Orchestrator Agent — 任务调度与用户交互入口
│
├── [数据参谋] Vault Agent — 用户债务数据管理与持久化
│
├── [算法核心] Optimizer Agent — WACC 计算与组合优化
│
├── [风控大脑] RiskGuard Agent — 压力测试 + 征信时序保护
│
├── [市场情报] Radar Agent — 平台利率数据库管理
│
└── [狂躁人格] Persona Agent — 最终输出格式化与语气注入
```

### 2.2 各 Agent 职责详述

#### Agent 1：Orchestrator（总司令）

**职责：**
- 接收用户所有输入
- 分解任务，分发给各子 Agent
- 收集各 Agent 结果，汇总后交给 Persona Agent 输出
- 管理对话状态与上下文记忆

**输入：**
- 用户的自然语言请求
- 触发类型识别（首次录入 / 更新数据 / 查询优化方案 / 查看还款日历）

**输出：**
- 结构化任务指令给各子 Agent
- 最终汇总结果给 Persona Agent

---

#### Agent 2：Vault（数据参谋）

**职责：**
- 维护用户的债务清单数据结构
- 负责数据的增删改查
- 在用户更新某笔负债时自动触发重新计算

**管理的数据结构（见第四章）：**
- 用户基本财务档案
- 债务记录列表
- 平台信息表（利率、规则）
- 历史优化方案记录

---

#### Agent 3：Optimizer（算法核心）

**职责：**
- 执行 WACC 计算
- 运行债务置换可行性判断
- 输出还款优先级排序
- 计算各方案的综合成本对比

**核心算法模块（见第五章）：**
- 加权平均资金成本计算
- 债务置换净收益不等式
- 还款次序优化（雪球法 vs 雪崩法动态切换）

---

#### Agent 4：RiskGuard（风控大脑）

**职责：**
- 运行蒙特卡洛现金流压力测试
- 管理征信申请时序规划
- 监控负债率红线
- 触发熔断机制

**核心算法模块（见第五章）：**
- 蒙特卡洛现金流模拟
- 征信查询冷却期计算
- DTI 预警系统
- 苟命协议（极端风险重组）

---

#### Agent 5：Radar（市场情报）

**职责：**
- 维护平台利率数据库（社区维护，手动更新）
- 接收用户手动录入的新优惠信息
- 对比当前负债与市场最优利率，输出置换机会列表

**说明：** MVP 阶段不做自动爬取，所有平台数据由用户或运营手动录入更新。

---

#### Agent 6：Persona（狂躁精算师）

**职责：**
- 接收其他 Agent 的结构化计算结果
- 转化为符合"狂躁精算师"人设的语言输出
- 根据情况严重程度调节"狂躁程度"

**情绪等级映射：**

| 情景 | 语气档位 |
|------|---------|
| 发现置换机会，用户可以省钱 | 亢奋、催促、用叹号 |
| 数据正常，无需操作 | 冷静报告，略带鄙视 |
| 用户操作有风险 | 咆哮模式，强烈警告 |
| 触发熔断，极端风险 | 严肃宣言，停止嬉皮笑脸 |
| 还款日临近 | 狂躁倒计时催促 |

## 三、核心功能模块

### 3.1 功能一：债务档案建立（首次录入）

**用户操作流：**

```
用户触发 →
Orchestrator 引导用户填写债务清单 →
Vault 存储结构化数据 →
Optimizer 计算初始 WACC →
RiskGuard 运行首次压力测试 →
Persona 输出"战场初勘报告"
```

**引导用户填写的最小必要字段（每笔负债）：**

1. 平台/机构名称
2. 当前余额（本金）
3. 年化利率（或月利率，自动换算）
4. 计息方式（等额本息 / 先息后本 / 随借随还）
5. 每月最低还款额
6. 剩余还款期数（或到期日）
7. 是否有提前还款违约金（及金额/比例）
8. 是否已上征信

**用户基本财务字段：**

1. 税后月收入（固定部分）
2. 月均浮动收入（可选）
3. 月均刚性支出（房租、生活费等，排除还款）
4. 可用现金储备（应急备用金）

---

### 3.2 功能二：WACC 优化计算与还款建议

**触发时机：** 用户每次更新数据后自动执行。

**输出内容：**

1. 当前综合资金成本（WACC）
2. 还款优先级排序（附理由）
3. 本月最优操作建议（还哪个？还多少？）
4. 预计 3 个月 / 6 个月 / 12 个月后的 WACC 趋势图（文字描述版）

**还款策略选择（用户可切换）：**

| 策略 | 适用场景 | 说明 |
|------|---------|------|
| 雪崩法 | 数学最优 | 优先还利率最高的，总利息最少 |
| 雪球法 | 心理激励 | 优先还余额最小的，快速清零建立信心 |
| 现金流保护法 | 现金紧张期 | 优先确保所有账单不逾期，其余资金保留备用 |

默认使用**雪崩法**，Agent 会解释为什么这个最省钱。

---

### 3.3 功能三：债务置换机会检测

**触发时机：**
- 用户手动触发"帮我找置换机会"
- Radar Agent 收到新的低息平台信息更新后自动触发检测

**判断逻辑（核心不等式）：**

```
可置换条件：
(旧贷款剩余利息总额) - (新贷款预计利息总额) - (手续费 + 违约金 + 新贷申请成本) > 用户设置的最低收益阈值

附加约束：
- 新贷款的月供不得超过当前月供的 120%
- 置换后的 DTI 不得超过安全阈值
- 征信查询次数未超过冷却期限制
```

**输出内容：**
- 置换可行性判断（可 / 不可 / 边界值需谨慎）
- 可节省的利息金额（明确的数字）
- 操作步骤建议（先做什么、后做什么）
- 风险提示（违约金窗口期、征信影响说明）

---

### 3.4 功能四：征信保护与申请时序规划

**核心原则：** 不帮用户"绕过"风控，而是帮用户**理性规划申请节奏**，避免密集硬查询引发拒批。

**用户输入：**
- 过去 6 个月内的征信硬查询记录（日期 + 机构）
- 计划申请的新贷款机构列表

**Agent 输出：**
- 当前查询频率评估（正常 / 偏高 / 高危）
- 建议的申请时间窗口
- 各机构的建议申请顺序（先申批准率高的，建立正向记录）
- 冷却期倒计时（建议何时恢复申请）

**内置查询频率安全阈值（默认值，可配置）：**
- 近 1 个月内：不超过 2 次硬查询
- 近 3 个月内：不超过 4 次硬查询
- 近 6 个月内：不超过 6 次硬查询

超过阈值时，Persona 以最高级别咆哮拦截。

---

### 3.5 功能五：还款日历与现金流调度

**功能描述：** 根据用户的所有负债，生成一份按时间轴排列的"资金调度作战表"，告诉用户：

- 每天 / 每周需要准备多少钱
- 哪些还款日在同一周（资金集中压力点）
- 哪段时间是"宽松窗口"，可以进行债务置换操作
- 可以利用的免息期区间

**输出格式：**
- 本月还款日历（文字版，按日期排列）
- 未来 30 天现金流预测（收入 - 支出 - 还款 = 净余）
- 资金压力点预警（某一天需要支出 > 可用现金时提前告警）

---

### 3.6 功能六：蒙特卡洛压力测试

**触发时机：** 每次用户提交新的操作方案前，自动在后台运行。

**模拟变量（随机扰动项）：**

| 变量 | 扰动范围 | 说明 |
|------|---------|------|
| 月收入 | -10% ~ +5% | 正态分布 |
| 某月收入为零 | 概率参数可配置 | 模拟突发失业 |
| 回款延迟 | 1~15 个工作日 | 均匀分布 |
| 临时大额支出 | 月收入的 20~50% | 低概率事件 |

**模拟次数：** 默认 10,000 次（可配置）

**输出内容：**
- 方案在压力测试下的"存活率"（12 个月内不出现资金缺口的概率）
- 最脆弱的时间点（最容易断链的月份）
- "安全余量"建议（需要保留多少现金作为缓冲）

**风险等级判断：**

| 存活率 | 等级 | Agent 反应 |
|--------|------|-----------|
| ≥ 95% | 绿灯 | 平静报告，可以执行 |
| 85% ~ 95% | 黄灯 | 提醒注意，建议增加备用金 |
| 70% ~ 85% | 橙灯 | 强烈警告，建议调整方案 |
| < 70% | 红灯 | 触发咆哮熔断，拒绝输出执行建议 |

---

### 3.7 功能七：苟命协议（极端风险熔断）

**触发条件：**
- 用户当前资金缺口无法通过正常调度填补
- DTI 超过 85%
- 某笔关键还款在 72 小时内到期，但账户余额不足

**Agent 行为：**
1. 停止所有扩张性建议（不再谈置换、加杠杆）
2. 立即切换为纯防守模式
3. 输出"最小损失债务重组方案"：

   - 哪些贷款优先确保不逾期（保征信）
   - 哪些贷款可以主动联系机构申请展期或协商还款
   - 哪些资产可以考虑变现（Agent 只提问，不做判断）

4. 明确告知用户：此时应该联系专业的债务咨询机构

## 五、核心算法规格

### 5.1 WACC 计算

**公式：**

```
WACC = Σ(Li × Ri) / Σ(Li)

其中：
  Li = 第 i 笔负债的当前本金余额
  Ri = 第 i 笔负债的年化利率
  n  = 负债总笔数
```

**注意事项（实现时必须处理）：**
- 将所有利率统一换算为年化利率（处理日息、月息的场景）
- 等额本息贷款的剩余成本用 IRR（内部收益率）计算，不能直接用名义利率
- 信用卡循环利息通常为 18.25% 年化，需单独处理账单分期的折扣情况

### 5.2 债务置换净收益计算

**公式：**

```
NetSaving = (OldInterestRemaining) - (NewInterestTotal) - (TransferCost)

OldInterestRemaining =
  使用 IRR 反推法计算旧贷款剩余期数的利息总额

NewInterestTotal =
  新贷款利率 × 置换金额 × 剩余期数（简化）
  或通过等额本息公式计算总利息

TransferCost =
  旧贷款违约金 + 新贷款手续费 + 征信查询机会成本（量化为分数）

可执行条件：
  NetSaving > MinThreshold（用户设置，默认 500 元）
  AND 置换后 DTI ≤ SafeDTI（默认 70%）
  AND 月供变化率 ≤ 20%
  AND 征信查询未超频
```

### 5.3 DTI 计算与预警

```
DTI = 月还款总额 / 月税后收入

安全阈值（默认，可配置）：
  绿灯：DTI ≤ 40%
  黄灯：40% < DTI ≤ 55%
  橙灯：55% < DTI ≤ 70%
  红灯：DTI > 70%  → 触发苟命协议评估
```

### 5.4 蒙特卡洛现金流模拟

**每次模拟的执行步骤：**

```
For 每次迭代（共 N=10000 次）：
  1. 生成未来 12 个月的随机收入序列
     Income[m] = FixedIncome × (1 + ε)，ε ~ N(0, 0.1)
     P(Income[m] = 0) = IncomeShockProb（默认 2%）

  2. 生成随机支出扰动
     ExtraExpense[m] ~ 泊松分布，低概率大额冲击

  3. 生成还款延迟扰动
     PaymentDelay[m] ~ Uniform(0, 5) 工作日

  4. 逐月计算现金流净额
     CF[m] = Income[m] - FixedExpense - Repayment[m] - ExtraExpense[m]

  5. 判断是否出现资金缺口（CF[m] < 0 且 CashReserve 被耗尽）

最终输出：
  SurvivalRate = 12 个月内未出现资金缺口的迭代次数 / N
  VulnerableMonth = 最多次出现缺口的月份
  RecommendedBuffer = 95th percentile 的最大月缺口金额
```

### 5.5 还款优先级排序（雪崩法）

```
对每笔债务计算"优先分数"：

PriorityScore =
  (AnnualRate × 0.5)                      // 利率权重 50%
  + (OverdueRisk × 0.3)                   // 逾期风险权重 30%
  + (PsychologicalWeight × 0.2)           // 余额权重 20%（辅助）

OverdueRisk = 1 / max(DaysToNextPayment, 1)  // 距下次还款越近越高

按 PriorityScore 降序排列，最高分优先还款
```

## 六、交互流程设计

### 6.1 首次使用流程

```
[用户] → "我想开始用通天贷"
[Orchestrator] → 触发引导录入流程

Step 1: 收集基本财务信息（月收入、月支出、现金储备）
Step 2: 逐笔录入负债（Vault 存储）
Step 3: Optimizer 计算初始 WACC
Step 4: RiskGuard 运行首次压力测试
Step 5: Persona 输出"战场初勘报告"

报告内容：
- 当前 WACC 是多少
- 危险程度（DTI 等级）
- 最紧迫需要处理的一笔债务
- 整体战略定性（进攻/防守/持平）
```

### 6.2 日常使用触发词（意图识别）

| 用户输入意图 | 触发动作 |
|------------|---------|
| "帮我算算现在的情况" | 运行全量分析，输出最新优化方案 |
| "有没有可以置换的" | 触发 Radar 对比 + Optimizer 置换计算 |
| "我想再借一笔" | RiskGuard 评估新增负债影响 + 征信时序建议 |
| "下个月的还款" | 输出近 30 天还款日历 |
| "我还了 XX，更新一下" | Vault 更新记录，重新计算 |
| "我现在很急，钱不够还" | 触发苟命协议评估流程 |

### 6.3 输出结构标准

每次输出必须包含以下层次：

```
1. [战情摘要] — 一句话说明当前最重要的事
2. [数字战报] — 关键指标（WACC、DTI、存活率）
3. [作战建议] — 本期最优操作（1~3 个步骤）
4. [风险提示] — 不可忽视的风险因素
5. [狂躁结语] — 符合人设的收尾语句
```

## 八、合规声明模板

每次输出方案时，结尾必须附加以下声明（可简化版本）：

```
⚠️ 以上内容仅为基于您提供数据的数学推算结果，
不构成任何金融投资或信贷决策建议。
最终决策请结合您的实际情况，必要时咨询持牌金融顾问。
```

**强制边界：**
- 不得建议用户提供虚假收入证明
- 不得建议用户伪造任何金融材料
- 不得建议用户使用任何未授权的自动化工具操作第三方平台
- 当用户提及"包装材料"类话题时，Agent 必须明确拒绝并提示风险
